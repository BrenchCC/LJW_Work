ADDA：
对抗判别式域适应

源域有标签、目标域无标签时，用对抗学习对齐特征空间，让分类器跨域迁移。

无监督域适应 对抗学习 特征子空间对齐
ADDA · Tzeng et al. · arXiv:1702.05464

今天分享的这篇论文是 ADDA，全称 Adversarial Discriminative Domain Adaptation。它解决的核心问题是：当你有源域标签、目标域没有标签时，怎么让模型在目标域上也能分类。

选这篇论文的原因跟我们正在做的工业缺陷检测直接相关。我们有五款电池的划痕标签，但第六款没有，直接迁移泛化性差。这个问题本质就是特征子空间的问题——产品外观域差异压过了缺陷语义。

ADDA 提供了一个清晰的对齐思路：用对抗训练把目标域特征拉到源域特征空间里，让源域分类器可以直接用。我会重点讲清楚它怎么做的、为什么有效、以及对我们项目的适用边界。

今天分享分成四个部分。先讲清楚域偏移问题是什么、为什么跟我们的缺陷检测痛点相关；然后展开 ADDA 的方法和训练流程；接着看实验证据支撑了什么结论；最后讨论它能不能用到我们的场景、有什么局限。

核心结论先剧透：ADDA 适合做我们技术路线里的域适应基线，但不能单独解决"第六款没有划痕样本"的根本风险。后面我会逐步展开为什么。

01 · the problem we face

我们的痛点：跨产品缺陷泛化差

前五款电池 有缺陷样本与标签

划痕、正常、其他缺陷类别提供监督信号。分类器在五款上表现很好。

→
直接迁移 泛化性明显下降

第六款电池的材质、光照、纹理变了，特征空间偏移，分类器失准。

→
第六款电池 没有划痕标签

暂时无法重新标注，需要无监督或半监督方法弥合域差距。

本质问题 产品外观域特征子空间压过了缺陷语义子空间；分类器学到的是"哪款产品"，而不是"有什么缺陷"。
Domain shift: product appearance dominates defect semantics

先把我们的问题说清楚。前五款电池有划痕标签，分类器训练出来效果不错。但第六款电池上线后，直接用前五款训练的模型，泛化性很差。原因是第六款的材质、光照、纹理变了，图像域发生了偏移。

这里的关键洞察是：分类器学到的其实不只是"划痕 vs 正常"，还学到了大量"这款电池长什么样"的信息。当产品外观变了，这些外观特征就变成了干扰。

所以本质上，我们要解决的是产品外观子空间和缺陷语义子空间的纠缠问题。ADDA 这类域适应方法，就是试图把两个域的特征分布拉近，让分类器不再被外观域差异干扰。

理想状态下，同类缺陷应聚在一起；但域偏移让特征按产品线分离，缺陷语义被淹没。

Same defect → different feature distribution → classifier fails

这页用图示把域偏移讲清楚。左边的源域是前五款电池，右边是第六款。划痕的物理本质是一样的——都是表面线性损伤，但在不同产品上的成像完全不同。材质反光不同、光照条件不同、相机位置不同，导致同一个缺陷在特征空间里落到了不同的位置。

下面的特征空间示意图说明了核心问题：源域特征和目标域特征形成了两个分离的簇。如果特征按产品线聚在一起，而不是按缺陷类型聚在一起，那分类器区分的其实是"哪款产品"而不是"有没有缺陷"。

所以域适应要做的事情就是：把目标域特征簇拉到源域特征簇附近，让"划痕"和"正常"的判别边界在目标域上也能生效。

03 · why this paper

ADDA 要解决什么

A

无监督域适应的经典设定

源域有标签 (Xs, Ys)，目标域只有图像 Xt，没有标签。希望模型在目标域上也能正确分类。

B

对抗学习对齐特征分布

训练一个域判别器区分源/目标特征，再训练目标编码器欺骗判别器，从而拉近两个域的特征分布。

C

判别式而非生成式

不生成目标域图像，只对齐特征空间。更轻量、更稳定，尤其适合分类任务。

Unsupervised DA · adversarial feature alignment · discriminative modeling

ADDA 的设定非常清晰：源域有标签数据，目标域只有无标签图像。这是一个非常典型的工业场景——我们有五款电池的划痕标签，第六款只有产线上的图像。

它的核心思路是用对抗学习做特征对齐。具体来说，训练一个域判别器来区分"这是源域特征还是目标域特征"，同时训练目标域编码器去欺骗这个判别器。如果判别器分不清了，说明两个域的特征分布已经足够接近。

ADDA 选择判别式方法而不是生成式方法，这是一个重要设计选择。它不尝试生成目标域图像，只做特征空间的对齐。这样计算量小，也不容易因为生成质量不好而引入额外噪声。

04 · framework overview

统一对抗域适应框架

四类设计选择

1. 基础模型：预训练网络初始化，还是从头训练？

2. 权重共享：源/目标编码器共享全部、部分、还是不共享权重？

3. 对抗损失：用什么方式做对抗——梯度反转、域混淆、还是 GAN loss？

4. 判别 vs 生成：只对齐特征，还是同时生成目标域图像？

ADDA 的选择

预训练初始化：用 ImageNet 预训练权重作为起点。

不共享权重：源编码器固定，目标编码器独立学习。允许目标域有自己的低层纹理映射。

GAN loss：目标编码器通过欺骗域判别器来学习对齐。

判别式：不生成图像，只做特征对齐，更适合分类任务。

关键设计 权重不共享 + GAN loss：目标编码器可以自由适应目标域的低层纹理，同时被约束在对齐的特征空间里。
Base model · weight sharing · adversarial loss · discriminative vs generative

ADDA 的一个贡献是它把现有方法放在统一框架下比较。域适应方法在设计上有四个关键选择：基础模型怎么初始化、源和目标编码器是否共享权重、用什么对抗损失、是判别式还是生成式。

ADDA 的选择是不共享权重加 GAN loss。不共享权重这一点很重要——共享权重意味着源和目标用同一套底层特征提取器，但不同产品的纹理差异可能需要不同的底层映射。让目标编码器独立学习，可以更好地适应目标域特有的低层纹理。

但这也意味着目标编码器需要从预训练权重开始"学会"一套新的映射，同时被对抗损失约束到源域特征空间里。这是一个在灵活性和约束之间的平衡。

05 · training pipeline

三阶段训练流程

1

源域预训练

用源域标签训练源编码器 Ms 和分类器 C。

目标：最小化分类损失 Lcls，让分类器在源域上表现好。

此阶段完成后，Ms 和 C 的参数固定。

2

对抗对齐

固定 Ms，训练域判别器 D 和目标编码器 Mt。

D 学习区分源/目标特征；Mt 学习欺骗 D。

对抗博弈让 Mt(Xt) ≈ Ms(Xs)。

3

目标域推理

测试时只用目标编码器 + 源分类器。

输入目标图像 → Mt 提取特征 → C 输出分类。

不需要判别器 D，也不需要目标域标签。

Stage 1: source pretrain · Stage 2: adversarial alignment · Stage 3: target inference

ADDA 的训练流程非常清晰，分三个阶段。第一阶段是源域预训练，用前五款电池的划痕标签训练编码器和分类器。训练完后编码器和分类器参数都固定下来。

第二阶段是核心：固定源编码器，训练域判别器和目标编码器。判别器要区分"这是源域特征还是目标域特征"，目标编码器要尽量让目标域特征被判别器误认为源域特征。这就是对抗博弈。

第三阶段就是推理。输入第六款电池图像，用目标编码器提特征，直接送进源域分类器出结果。判别器只在训练时用，推理时不需要。整个流程不需要第六款的任何标签。

06 · architecture diagram

ADDA 架构图

关键观察

Source CNN 训练后固定（虚线），Target CNN 单独学习自己的低层映射。

域判别器 D 只在第二阶段参与，推理时不需要。

对应到电池场景

Source CNN = 前五款电池特征提取器；Target CNN = 第六款电池特征提取器。

分类器 C 复用前五款的缺陷分类头，不需要重新训练。

Figure 3 from the paper — source encoder (fixed) · target encoder (learned) · domain discriminator

这页是论文中的架构图。左边是源域：带标签的图像经过 Source CNN 提取特征，然后送给分类器。中间是目标域：无标签图像经过 Target CNN，特征要和源域特征混在一起骗过判别器。

注意虚线表示固定参数——Source CNN 在第一阶段训练完后就不变了。这是 ADDA 的一个设计选择：源编码器不动，让目标编码器来适应源域空间。

映射到我们的场景：前五款电池就是源域，第六款就是目标域。分类器 C 是前五款训练好的缺陷分类头，直接复用。关键问题在于：对抗对齐能否把第六款的特征拉到前五款的特征空间里？

07 · math

三个优化目标：分类 + 判别 + 欺骗

① 分类损失 min L_cls(X_s, Y_s)
② 判别器损失 min L_advD = -E[log D(M_s(x_s))] - E[log(1-D(M_t(x_t)))]
③ 目标编码器损失 min L_advM = -E[log D(M_t(x_t))]
Classification → Discrimination → Fooling — standard GAN formulation for feature alignment

这页是 ADDA 的三个优化目标。第一个是分类损失，用源域标签训练源编码器和分类器，让它在前五款电池上学会区分划痕和正常。

第二个是判别器损失，标准 GAN 判别器目标——让它正确区分源域特征和目标域特征。

第三个是目标编码器损失，这是对抗的部分——让目标编码器生成的特征被判别器误认为是源域特征。如果判别器分不出来了，说明两个域的特征分布已经足够接近。

第三个目标是整个方法的核心。它不要求目标域有标签，只要求目标域有无标签图像。这正是我们的场景：第六款电池虽然没有划痕标签，但有产线图像可以用。

08 · intuition

为什么对抗对齐有效

源域特征
目标域特征（对齐后） ←

对齐前

源域和目标域特征分布差异大

源域分类边界在目标域上失效

模型预测塌缩：全部预测为多数类

对齐后

目标域特征被拉到源域特征空间

源域分类边界可以继续使用

预测分布更分散，更接近有监督结果

核心逻辑 如果 M_t(X_t) ≈ M_s(X_s)，那么源域学到的分类器 C 就能直接用到目标域特征上。
Feature alignment → shared classifier → cross-domain prediction

这页解释为什么对抗对齐能工作。左边的图展示了特征空间的变化：对齐前，源域和目标域特征是分离的；对齐后，目标域特征被"拉"到了源域特征空间里。

对齐前，源域分类器学到的判别边界在目标域特征上是失效的——因为目标域特征落在边界之外。一个常见症状就是预测塌缩：模型把所有目标域样本都预测为同一个类。

对齐后，目标域特征落在了源域特征空间里，源域分类器的判别边界就可以继续使用了。这就是 ADDA 的核心逻辑：不重新训练分类器，而是让目标域特征去"适应"已有的分类器。

09 · design choice

为什么不共享权重

共享权重的局限

源和目标用同一套底层特征提取器

底层特征被"绑死"，无法适应目标域特有纹理

当域偏移较大时，共享底层可能反而限制对齐

代表方法：DANN (梯度反转)、Domain Confusion

不共享权重的优势

目标编码器可以学习目标域特有的低层映射

源编码器固定，保留源域判别性特征

对抗约束只作用于高层特征空间

在 SVHN → MNIST (大域偏移) 上明显更稳定

电池缺陷场景的意义

不同电池型号的表面纹理差异可能很大——光泽度、颗粒感、反光模式都不同。让目标编码器独立学习底层映射，可以更好地捕捉第六款电池特有的纹理特征。

Shared weights: simpler but rigid · Separate weights: flexible but needs careful init

ADDA 选择不共享权重，这个设计需要特别解释一下。共享权重的方法比如 DANN，让源和目标用同一个编码器，通过梯度反转层来对齐特征。好处是简单，但问题是底层特征被绑死了，两个域必须共用同一套底层映射。

当域偏移不大的时候，共享权重没问题。但当域偏移较大——比如 SVHN 迁移到 MNIST，或者前五款电池迁移到第六款——底层纹理差异可能大到共享权重反而限制了对齐效果。

ADDA 让目标编码器独立学习，对抗约束只作用在高层特征空间。这意味着目标编码器可以自由地学习第六款电池特有的底层纹理映射，同时保证高层特征和源域对齐。这是一个更灵活的设计。

10 · prerequisite

ADDA 的前提条件：不是万能的

✓

前提 1：目标域有无标签图像

ADDA 必须有目标域图像 Xt 来训练目标编码器和判别器。

如果第六款电池连无标签图像都没有，ADDA 无法启动。

?

前提 2：目标域图像包含缺陷样本

ADDA 对齐的是边缘分布 P(X)，不是条件分布 P(Y|X)。

如果目标域只有正常样本，划痕子空间可能无法被正确对齐。

!

前提 3：源/目标类别空间一致

ADDA 假设源域和目标域共享相同的类别标签集合。

如果第六款出现了前五款没有的缺陷类型，ADDA 无法处理。

风险提醒 如果第六款只有正常样本，ADDA 可能只对齐正常表面纹理，无法保证划痕子空间被正确迁移。
ADDA aligns marginal P(X), not conditional P(Y|X) — this is the key limitation

这页很重要，讲 ADDA 的前提条件。很多论文汇报只讲方法多好，不讲什么时候不能用。对我们的场景来说，这个前提检查非常关键。

第一个前提：目标域必须有无标签图像。如果第六款电池连产线图像都没有，ADDA 根本没法启动。不过在我们的场景里，这个条件是满足的——第六款电池有产线正常品图像。

第二个前提是核心风险：ADDA 对齐的是边缘分布，也就是 P(X)，而不是条件分布 P(Y|X)。它保证的是"两个域的特征分布整体接近"，但不保证每个类别分别被正确对齐。如果第六款的无标签样本里主要是正常品，对抗对齐主要学到的就是正常表面纹理的对齐，划痕子空间可能被忽略。

第三个前提是类别空间一致。如果第六款出现了全新的缺陷类型，ADDA 也处理不了。

11 · evidence

实验 1：数字数据集跨域迁移

90.1% USPS → MNIST (ADDA)
89.1% USPS → MNIST (CoGAN)
76.0% SVHN → MNIST (ADDA)
— SVHN → MNIST (CoGAN 未收敛)
关键结论 域偏移越大（SVHN → MNIST），判别式方法越稳定；生成式方法 (CoGAN) 在大偏移下可能不收敛。
MNIST · USPS · SVHN — discriminative > generative under large domain shift

第一个实验是数字数据集上的跨域迁移。USPS 到 MNIST 的域偏移比较小，ADDA 达到 90.1%，比 CoGAN 的 89.1% 略好。但更关键的对比是 SVHN 到 MNIST——这里域偏移很大，SVHN 是真实场景的街景数字，MNIST 是手写数字。

在这个大偏移场景下，ADDA 达到 76.0%，而 CoGAN 直接没有收敛。这说明生成式方法在大域偏移下可能不稳定，而判别式方法更鲁棒。

映射到电池场景：前五款到第六款的域偏移可能更接近 SVHN 到 MNIST 的情况——不同产品的外观差异可能相当大。所以 ADDA 的判别式设计在稳定性上有优势。

第二个实验是 NYUD 数据集上的跨模态分类，从 RGB 图像迁移到深度图。这是论文中最接近工业跨域的证据——RGB 和深度图的成像原理完全不同，就像不同电池型号的表面纹理差异。

Source only 只有 13.9% 的整体准确率，ADDA 提升到 21.1%，提升了 7 个百分点以上。在完全没有目标域标签的情况下，这是很有意义的提升。

但需要注意论文的诚实之处：并非所有类别都提升。某些类别在域适应后仍然无法恢复。这一点对电池划痕检测尤其重要——如果某些缺陷类型在目标产品中没有覆盖，域适应不一定能凭空恢复。

这页是核心结论页。ADDA 能用到我们的电池缺陷检测吗？答案是：可以用作域适应基线，但不能单独解决所有问题。

具体来说，如果我们把前五款电池作为源域，第六款的无标签产线图像作为目标域，ADDA 可以削弱产品外观域偏移。而且它不需要第六款的标签，只需要产线正常品图像就能启动。

15 · proposed solution

更稳妥的组合方案

1 多源缺陷库

五款电池作为多源域，提供更丰富的缺陷特征覆盖。

2 ADDA 域适应

用对抗对齐削弱产品外观域偏移，作为域适应基线。

3 类条件约束

加入原型约束或 MMD/CORAL，保证划痕子空间也正确对齐。

4 少量主动标注

对第六款少量高不确定性样本做人工标注，弥补条件分布对齐不足。

5 异常检测兜底

用正常样本 PCA 残差或 Mahalanobis 距离做异常检测，覆盖未知的缺陷类型。

核心策略 ADDA 做域适应基础层，类条件约束保缺陷子空间，少量标注补条件分布，异常检测兜底未知类。
Multi-source + ADDA + conditional constraint + active labeling + anomaly detection

既然 ADDA 不能单独解决问题，那应该怎么组合？我建议一个五步方案。

第一步，把五款电池作为多源域。不是只用一款作源域，而是全部用上，提供更丰富的缺陷特征覆盖。不同电池的划痕形态可能有差异，多源域可以覆盖更多变化。

第二步，用 ADDA 做域适应基线。这是最直接的——用对抗对齐削弱产品外观域偏移。

第三步是关键改进——加入类条件约束。ADDA 只对齐边缘分布，我们需要额外约束来保证划痕子空间也被正确对齐。方法可以是原型约束、MMD、CORAL 等。

第四步，少量主动标注。选择第六款中模型最不确定的少量样本做人工标注，弥补条件分布对齐的不足。这比全量标注成本低很多。

第五步，异常检测兜底。用正常样本建模做异常检测，覆盖可能出现的未知缺陷类型。

16 · summary

总结：ADDA 的定位与边界

ADDA 做对了什么

清晰的无监督域适应框架：源域预训练 → 对抗对齐 → 目标推理

判别式方法比生成式更稳定，尤其是大域偏移下

不共享权重设计允许目标域有独立低层映射

作为域适应基线，方法简洁、实验充分

ADDA 的局限

只对齐边缘分布，不保证类别条件对齐

目标域无标签数据中必须有缺陷样本覆盖

不能处理开放集或新类别场景

不适合单独作为工业缺陷迁移的最终方案

结论 ADDA 适合做域适应基线——削弱产品外观偏移；但划痕子空间对齐需要额外约束（类条件 + 少量标注 + 异常检测）。
Good baseline · not the final answer · combine with conditional alignment + active labeling + anomaly detection

最后做一个总结。ADDA 做对了几件事：提供了清晰的三阶段框架、判别式方法比生成式更稳定、不共享权重设计更灵活、作为基线方法简洁有效。

但它也有明确的局限：只对齐边缘分布不保证类别条件对齐、需要目标域有缺陷样本覆盖、不能处理新类别。

所以我的结论是：ADDA 适合做我们技术路线中的域适应基线，用来削弱产品外观偏移。但划痕子空间的对齐需要额外手段——类条件约束、少量主动标注、异常检测兜底。这个组合方案才是更稳健的工业级解法。
